## 介紹定義
在電腦科學中，樹（Tree）是一種非常常見的資料結構，它是由若幹個節點（Node）組成的，這些節點之間存在一些特定的關係。

樹的節點可以看作是一個個資料元素，而樹的關係則是這些資料元素之間的邏輯關係。

樹的每個節點可以有零個或多個子節點，一個節點沒有父節點的節點稱為根節點（Root），一個節點沒有子節點的節點稱為葉子節點（Leaf）。

樹的每個非根節點都有一個父節點，每個節點都可以有任意多個子節點。

樹的深度（Depth）是指從根節點到葉子節點的最長路徑，樹的高度（Height）是指從葉子節點到根節點的最長路徑。

## 應用
樹的應用非常廣泛，例如檔案系統、資料庫索引、編譯器語法分析等都可以使用樹來實現。

常見的樹結構包括二元樹（Binary Tree）、AVL樹、紅黑樹（Red-Black Tree）等。

- 二元樹是一種特殊的樹結構，它的每個節點最多有兩個子節點，稱為左子節點和右子節點。二元樹的應用非常廣泛，例如二叉搜尋樹（Binary Search Tree）可以用來實現快速尋找、插入和刪除等操作，堆（Heap）可以用來實現優先佇列等。

- AVL樹是一種自平衡的二叉搜尋樹，它的每個節點都有一個平衡因子（Balance Factor），用來表示左子樹的高度和右子樹的高度之差。通過旋轉操作來保持樹的平衡，AVL樹可以保證在最壞情況下的尋找、插入和刪除操作的時間複雜度都是O(log n)。

- 紅黑樹是一種自平衡的二叉搜尋樹，它通過保持以下性質來保持平衡：每個節點都是紅色或黑色；根節點是黑色的；每個葉子節點都是黑色的空節點；如果一個節點是紅色的，則它的子節點必須是黑色的；從任意節點到其每個葉子節點的所有路徑都包含相同數目的黑色節點。通過旋轉和顏色變換操作來保持樹的平衡，紅黑樹可以保證在最壞情況下的尋找、插入和刪除操作的時間複雜度都是O(log n)。

總的來說，樹是一種非常重要的資料結構，它在電腦科學中有著廣泛的應用，可以用來解決各種問題。不同的樹結構有不同的特點和應用場景，選擇合適的樹結構可以提高演算法的效率和性能。

## 常見10種樹及其特點的表格：

| 樹結構 | 特點 |
| --- | --- |
| 二叉樹（Binary Tree） | 每個節點最多有兩個子節點，左子節點和右子節點。 |
| 二叉搜索樹（Binary Search Tree） | 一種特殊的二叉樹，左子節點的值小於父節點的值，右子節點的值大於父節點的值。可以用來實現快速查找、插入和刪除等操作。 |
| 堆（Heap） | 一種特殊的樹結構，可以用來實現優先隊列等操作。堆分為最大堆和最小堆兩種類型，可以用來實現快速的插入、刪除和查找最大/最小值等操作。 |
| 平衡二叉樹（Balanced Binary Tree） | 一種高度平衡的二叉樹，可以保證查找、插入和刪除等操作的時間複雜度都是O(log n)。常見的平衡二叉樹包括AVL樹和紅黑樹。 |
| B樹（B-Tree） | 一種多路搜索樹，每個節點可以有多個子節點。B樹通常用於文件系統和數據庫索引等場景。 |
| B+樹（B+Tree） | 一種基於B樹的改進，所有數據都存儲在葉子節點中，非葉子節點只存儲索引信息。B+樹通常用於數據庫索引等場景。 |
| Trie樹（字典樹） | 一種特殊的樹結構，用於字符串的查找和匹配。Trie樹的每個節點都表示一個字符串的前綴，可以用來實現快速的字符串匹配操作。 |
| 伸展樹（Splay Tree） | 一種自適應的二叉搜索樹，可以通過旋轉操作來保持樹的平衡。伸展樹會根據最近訪問的元素來調整樹的結構，可以提高查找效率。 |
| 哈夫曼樹（Huffman Tree） | 一種用於數據壓縮的樹結構，可以根據字符出現的頻率來構建最優的編碼表。哈夫曼樹通常用於無損壓縮算法中。 |
| 線段樹（Segment Tree） | 一種用於區間查詢的樹結構，可以用來實現區間最大值、最小值、求和等操作。線段樹通常用於解決一些區間查詢問題。 |

以上是常見的10種樹及其特點的表格，這些樹結構在計算機科學中都有著廣泛的應用，可以用來解決各種問題。選擇合適的樹結構可以提高算法的效率和性能。
